HTTP Catcher过滤器改成圈x规则

发表于2021-04-21|更新于2021-04-22|网络HTTP Catcher
|字数总计:2.2k|阅读时长:7分钟|阅读量:

获得一个可用的httpcatcher过滤器（hcc或conf文件）

过滤器的获得主要有2种途径。

第一种：自己抓包自己写一个

第二种：找大佬分享的。
从大佬那里获取的加密过滤器
很多大佬出于版权考虑，
会给过滤器加密，
这时你只能使用，但没办法查看具体的修改内容。
这里就需要大家打勾过滤器后，
运行app，在确定修改生效后，
回去httpcatcher的抓包结果中，
翻阅数据包，查看被修改的内容，
从而逆向反推出作者修改了什么。
只要有耐心，并不难找，
技巧就是对比开启和关闭过滤器后数据的不同，仅此而已，也就费点头发……

图解httpcatcher过滤器数据的分析

  • 1、首先我们从AppStore下载黑岩小说
  • 2、打开app是正常的普通账号状态
  • 3、然后我们打开由搞机大神友情赞助的黑岩小说过滤器
  • 4、打勾过滤器并启用httpcatcher后，App中账号信息已变化，说明有效。
  • 5、我们来分析源码，直接选择在文本编辑器编辑
  • 6、文本编辑器中，可以用文字描述将代码分割成2部分来分析。文字说明上方，是文本修改内容
  • 7、文本下方，是修改数据路径内容

8、我们先分析下半部分，这里最重要的数据有3个：

（1）scheme，也就是协议，主要是判断为http还是https。
上图中明显为https。

（2）host ，也就是域名，
可以从上图中看到，本过滤器中出现了两个host，但内容是一样的，我们可以直接记为 hostname = biz.lemonyd.com。
咦，是不是有圈X那味道了？

（3）path ，也就是路径 ，他和域名加在一起其实就是指向这个数据包存在的位置。上图中有2个path 且内容不同，说明这个过滤器修改了2个不同位置的数据包。
这3个重要数据拼接起来，其实就是一个完整的链接，比如上面可以拼成： https://biz.lemonyd.com/user/account/info 和 https://biz.lemonyd.com/activity/activity/account

由此我们知道了，这个过滤器是通过修改上面2个路径的数据包而得以实现的。

9、我们分析上半部分数据，看改了什么。如图可以看到，这里的数据是由多个格式重复的大括号构成的，区别只有几个数值不同

由此可以轻易知道，我们需要用到的关键值主要是2个，另外还有一些用于判断的值我们放在后面再说：

（1）matchvalue：即匹配的值，也就是原数值、没改过的数值的意思。
红框括号中，matchvalue的值是 isVip” : \w+
这里的 \w+ 是正则表达式，意思是匹配任意多个字母串，
也就是说，这里的意思是：
没改过的正常数值为：isVip” : 某些字母。
同理，绿色括号中的 \d+， 在正则中表示 匹配任意多个数，
也就是，这里的的意思是：
没改过的正常数值为：shell”:某个不知道具体多少位的数字。
是不是懂了？

（2）Value ：即数值，这里的意思是改后的数值、最终的数值。
红框括号中，Value的值是 isVip” : true
结合上一步的意思一起理解就很容易了，
就是：
无论isVip” : 后面的字母是什么，一律修改成isVip” : true
同理可知绿色括号中的意思是：
无论shell” : 后面的数字是什么，一律修改成shell” : 6666666
至此，我们已经完全理解了httpcatcher是如何运作修改数据包的。

10、首先理解，圈x的修改规则是如何运作的？

圈x的修改规则，由2个部分构成，分别是：
（1）mitm，也就是Man-in-the-MiddleAttack，
俗称第三者中间人攻击。
圈x中有[mitm]标签，用过的都清楚。
在这个标签下填写一个hostname，
圈x就会劫持往来于这个域名的所有数据包，
并让本地用户数据认为圈x是服务器，
让服务器认为圈x是用户。
圈x就可以在中间上下其手、吃完上家吃下家。

（2）rewrite，俗称重写，也就是改写数据包。
毕竟圈x劫持了数据是要爽一下的嘛，否则不就白抓了小白羊了~
当上述2者完美匹配的时候，圈x的修改规则就生效了。

11、圈x如何从httpcatcher中获取[mitm]的值？

这里比较简单，
上一章讲httpcatcher时，我们已经看到了，过滤器中有host这个关键数据。这里就是通用的。

在此我简单展开点几个知识点，大家可以在日后举一反三：
（1）httpcatcher过滤器中只有单一的host时。
这种最容易，比如我们今天举例的黑岩小说就是这样，此时直接就可以照搬：hostname = biz.lemonyd.com

（2）httpcatcher过滤器中有多个host时。
偷懒的办法就是全都罗列起来。比如黑岩如果不只一个域名，而是4个不同的域名时，我们就可以直接堆成下面这样：hostname = biz.lemonyd.com,biz.lemonyd2.com,biz.lemonyd3.com,biz.lemonyd4.com,

这里注意，多个域名直接用英文的逗号隔开，注意一定用英文。

（3）httpcatcher过滤器中有多个host时，
也可以优雅的使用正则表达式来实现，如上一步的4个域名，可以直接写作：hostname = biz.*.com 效果相同

12、圈x的重写需要从httpcatcher中获取什么？

圈x的重写由3个部分构成，
我随便复制一个之前写的七喵小说的规则，
https://(.+).wtzw.com/.+ url script-response-body sngxqmxs.js

可以看到，3部分分别是：

（1）指向脚本位置的路径：
https://(.+).wtzw.com/.+

（2）申明想要修改什么：
url script-response-body （url反馈回的消息体）

（3）怎么修改：
sngxqmxs.js ，这里是指向了一个js文件，里面写好了修改规则
所以，我们想要改写hcc为圈x规则，就要获取上面的3个部分。

13、圈x怎么从httpcatcher中获取上面的3要素？

（1）申明修改内容：
这个最简单，看httpcatcher过滤器中的action和destiontion就知道了。

如图，可以明显看到，是response和body。
因此这里我们沿用 url script-response-body 就可以了。
事实上，9成的修改规则都是修改反馈的消息体，
其他情况都是少数的，那些情况以后我们有机会再聊，
今天主要就是给大家领进门，就不说太复杂的了。

（2）指向脚本位置的路径：
刚才在httpcatcher的分析中，我们已经得出了路径，那就是下面2条。https://biz.lemonyd.com/user/account/info 和 https://biz.lemonyd.com/activity/activity/account

在圈x中为了引用方便，一般都是用正则表达式将路径合并起来。
我们将上面2个路径合二为一：https://biz.lemonyd.com(\/user\/account\/info|\/activity\/activity\/account)

合并方法应该一看就明白，前面相同的部分不动，
后面不同的部分用括号包起来，中间用一个 | 隔开。
注意：所有符号都必须是英文。
至此，路径问题也解决了。

（3）最关键的问题就是修改问题了，
为了方便大家自己做，这里我直接为大家提供一个模板，大家照着抄作业就行了

body = $response.body.replace(/matchvalue/g, 'value')..replace(/matchvalue/g, 'value');
$done({body});

将 httpcatcher 中的 matchvalue、vaulue的值，
替换上面模板中的对应位置就行了。
还是以黑岩为例，替换后就是如下的代码了：

body = $response.body.replace(/isVip":\w+/g, 'isVip":true').replace(/shell":\d+/g, 'shell":6666666');
$done({body});

将其 保存为js 文件，命名为hyxs.js即可。

14、如何引用改好的圈x规则？

从3-4步，已可获得完成的圈x规则为：
[mitm]

Hostname = biz.lemonyd.com

[rewrite local]

https://biz.lemonyd.com(\/user\/account\/info|\/activity\/activity\/account) url script-response-body hyxs.js

我们只需要把 xhyxs.js 文件 保存到圈x 本地，

然后把上面2条规则粘贴到圈x配置文件的对应位置就可以了。

当然，这里也可也将文件上传到远程仓库如github，然后用远程地址来加载，这都是基础知识不再赘述。

都添加后，断开圈x并重连，再重新开启黑岩小说app，发现效果和开httpcatcher一样了。
